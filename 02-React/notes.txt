- earlier we saw how we injected react using CDN link that was the one way

- another way to get react into our app is via npm 
- what is react in the end of the day ?it's just a js code and it is hosted over npm also

- in npm we get React from npm now we don't want to use CDN link this is not a good things to bring the React into our project

Why?
- it is a costly operation if we fetch from CDN it will make a network call to unpkg.com and it will get react from unpkg.com 
suppose if i already have React in our node module how easy it would be to use react isn't?

- we don't want to make another network call to react  instead we will have it in node_modules

- 2nd thing is if someday a new version came then we need  to update the CDN link with the new version so it's better to have it inside package.json it easier to manager all our dependencies and is it very easy to manage react also as one of the dependencies  inside our npm package.json
